How to build a complete database — run order guide

Each phase depends on the one before it. Run top-to-bottom for a full rebuild; for routine weekly updates only phases 2–4 are needed.

1
Voter File — the foundation everything joins to
Run once per NYS file release (~every few months). Loads 13 M voter records and computes name-cleaning columns used by all matching passes.
Main tab → Pipeline
2
Donor Data — enriches voter_file rows with giving history
BOE, National, and CFB are independent — run in any order. Each adds donation columns (boe_total_amt, cfb_total_amt, etc.) to voter_file that are then copied onto matched CRM contacts.
Donors → BOE: Download → Load & Classify → Enrich VF
Donors → National: Download → Extract → Load & Classify → Enrich VF
Donors → CFB: Download → Load & Classify → Enrich VF
3
CRM Sync — pull contacts from HubSpot & Campaign Monitor
Merges all contact sources into crm_unified.contacts. Must run before matching so there are contacts to match against.
CRM tab → All CRM
4
CRM → Voter Matching — run in this exact order
Each pass only processes contacts the previous pass missed. Running out of order wastes time but causes no data errors.
① Match to Voter File
Exact name + zip. Covers ~60–70% of contacts.
CRM tab → Match to Voter File
② Phone Match
Phone number lookup against voter_file. Catches ~5–10% more.
CRM tab → Phone Match
③ Extended Match
Hyphenated surnames, compound first names, inactive voters.
CRM tab → Extended Match
5
Derived Enrichment — computed fields on matched data
Runs last because it depends on voter match results and donor data both being present.
Enrich tab → Ethnicity Enrich tab → Derived Fields Enrich tab → District Scores
Routine updates (weekly): Only phases 2–4 are needed — all steps are incremental and skip unchanged data automatically. The voter file pipeline (phase 1) only needs re-running when NYS publishes a new extract.

BOE — NY State Campaign Finance

NY Board of Elections bulk filings: state & county candidate and committee contributions. Uses a Playwright headless browser to bypass the BOE download portal. Steps: Download ZIPs → Load into DB & classify parties → Enrich voter file.

1Download
Scrapes BOE site via Playwright and saves 4 bulk ZIP files (~200–400 MB each). Skips unchanged files unless forced.
2Load & Classify
Loads ZIP contents into boe_donors DB, then classifies each contribution as D/R/U using filer keywords and candidate cross-reference.
3Enrich Voter File
Merges boe_donor_summary into voter_file — writes BOE total/D/R amounts, counts, last gift date and filer per voter.
National Donors — Federal Campaign Finance

Federal Election Commission individual contribution files — 6 election cycles (~10 years). Each step is independent so you can resume from any point if a step fails. Steps: Download & Extract → Load & Classify → Enrich voter file.

1Download & Extract
Downloads 6 cycles of federal bulk ZIPs from fec.gov, extracts them immediately, then deletes the ZIPs. Hash-checked — skips unchanged cycles.
2Load & Classify
Bulk-loads federal .txt files into National_Donors DB via LOAD DATA INFILE, then classifies ~6K committees as D/R/Unknown.
3Enrich Voter File
Matches national contributions to voter_file by name+zip, writes national donation totals, D/R amounts, and is_national_donor flag.
CFB — NYC Campaign Finance

NYC Campaign Finance Board individual contributions across 4 election cycles (2017, 2021, 2023, 2025). Steps: Download CSVs → Load & Enrich voter file (CFB load also writes directly to voter_file).

1Download
Downloads 4 CSV files from NYC CFB data library. Uses HTTP HEAD + MD5 to skip unchanged cycles.
2Load & Enrich
Loads CSVs into cfb_donors DB and matches contributions to voter_file by name+zip. Writes per-cycle amounts and totals. Force-reload re-processes even if file hash is unchanged.
