Phase 2 Donor Data — enriches voter_file rows with giving history

BOE, National, and CFB are independent — run in any order. Each adds donation columns (boe_total_amt, cfb_total_amt, etc.) to voter_file that are then copied onto matched CRM contacts.

BOE + National + CFB in sequence — skips downloads, uses existing files
Re-downloads all source files first, then processes all three
BOE — NY State Campaign Finance
1Download
Scrapes BOE site via Playwright — saves 4 bulk ZIP files + COMMCAND filer list (~200–400 MB each).
2Load & Classify
Loads into boe_donors DB, classifies each contribution as D/R/U using filer keywords and candidate cross-reference.
3Enrich VF
Merges boe_donor_summary into voter_file — writes BOE total/D/R amounts, counts, last gift date per voter.
National Donors
1Download & Extract
Downloads 6 cycles of federal bulk ZIPs from fec.gov, extracts immediately, deletes ZIPs. Hash-checked — skips unchanged.
2Load & Classify
Bulk-loads federal .txt files into National_Donors via LOAD DATA INFILE, classifies ~6K committees as D/R/Unknown.
3Enrich VF
Matches national contributions to voter_file by name+zip, writes donation totals, D/R amounts, and is_national_donor flag.
CFB — NYC Campaign Finance
1Download
Downloads 4 CSV files from NYC CFB data library. Uses HTTP HEAD + MD5 to skip unchanged cycles.
2Load & Enrich
Loads CSVs into cfb_donors DB and matches to voter_file by name+zip. Writes per-cycle amounts and totals.
Advanced: Data Reset

Destructive operation. Drops the boe_donors, National_Donors, and cfb_donors databases and NULLs all enrichment columns on voter_file. Use DB only to drop and clear without re-running — useful before a full re-download.

DB only — drops databases and clears voter_file columns, stops there.  |  Drop & Rebuild — drops then re-runs the full donor pipeline (very long).
Phase 5 Derived Enrichment — computed fields on matched data

Runs last because it depends on voter match results and donor data both being present. Adds computed and modelled columns directly to the voter file.

Build Ethnicity Model
BISG name-probability model predicts ethnicity for every voter. First run takes 30–60 min. Subsequent runs fill only NULLs.
Batch size
Enrich Derived
Computes registration recency, turnout scores, donor cross-level flags, and household clustering. Default fills NULLs only.
District Scores
Builds the district_scores table — partisan lean, competitiveness category (Safe D → Safe R), average turnout, and donor participation rate per CD/SD/LD.
Party Snapshot
Records today's party registrations and compares to the previous snapshot to detect switchers. Stamps is_party_switcher, prior_party, and party_change_date.
